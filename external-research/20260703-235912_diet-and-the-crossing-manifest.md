## Part One — What We Mean by Diet

### The problem diet solves

An autonomous agent with repository access can always find *something* to do. The tree is large; curiosity is cheap; and an hour of confident motion without a ruling is how policy drifts, names appear unbidden, and parity red gets explained away in prose. **Diet** is the positive answer: not "do not wander," but **here is the named menu you may eat this minute** — every item precondition satisfied, every owner tag yours, every gate already open.

The word arrived in the Cursor loop (`20260702.211046`) beside three companions that complete it:

| Term | One line |
|------|----------|
| **Gate** | A precondition owned by someone else or an unlanded event — a `(Kaeden …)` box, a door ruling, a paste not yet arrived, counsel not yet confirmed |
| **Ungated** | Every precondition for the next bite is satisfied **now**; the owner tag is Cursor (or the tagged autonomous sitter) |
| **Diet** | The **named menu** of tasks the sitter may execute **without** simulate/wait/ask — never a foraging license |
| **Smallest-first** | Order **within** the diet — the smallest witness or doc wins the next round |
| **Witness-first, red-then-green** | Method **within** the diet — prove on metal before claiming green; fail loudly, then land in the same sitting |
| **Empty plate** | When the menu runs out — **stop and summarize**; an empty plate is a question for Kaeden, never permission to invent |

Read together: **diet** is not ambition, it is **bounded appetite**. The recursion prompts, TASKS, and RTAC hold the menu; the agent eats one item, witnesses, commits, and returns for the next menu line — or stops.

### What diet is not

Diet is **not** the whole roadmap. ROADMAP holds the why and the order across seasons. Diet is **this sitting's** slice.

Diet is **not** counsel. External-research and Claude surveys **propose**; they do not enter the menu until seated or ratified.

Diet is **not** foraging. "The tree probably needs X" is not on the menu unless X was named when the plate was set.

Diet is **not** mood. "Ungated diet: RW-2 season" is a **list**, not a vibe — if RW-2 is checked and parity green, the plate is empty until Kaeden or TASKS names the next item.

### How diet moves through a sitting

A typical arc — the one this week demonstrated:

1. **Menu declared** in RTAC (`chrono sweep wire · RW-2 season · …`).
2. **Smallest-first** picks chrono wire (one parity block) before RW-2 (two witnesses + build).
3. **Witness-first** runs parity red if needed, then green, then commit.
4. **Plate refresh** — TASKS and RTAC update; crossed items leave the menu.
5. **Empty plate** or **Kaeden word** opens the next season.

Counsel rounds (bench-as-block, crossing suite) **sit outside** diet unless explicitly seated — they are Claude's lane; Cursor intakes when packets land.

**Diet** reads well beside **gate**, **ring**, **green**, **season**, **bench** — all short, all bodily. The risk: it sounds like nutrition or restriction rather than **authorized menu**. Hence candidates below.

## Part Two — The Crossing Manifest Seed

### Filename honesty

The artifact today is **`work-in-progress/20260703-202312_crossing_manifest_seed.rye`** — not yet `crossing_manifest.rye`. The `_seed` suffix means: **proposed, unverified, no bench claim**. A witness must run red-then-green before the name graduates. Ring one of Amphora formalizes the **manifest line**; the full vessel (zip, seal, stamp, shoulder) is larger than this seed.

### What problem it solves

Every night the Rio↔Cursor handoff improvises a container:

- Zip file, additions-only
- Parent git hash in the archive comment
- File list carried by convention and prose
- Rules re-taught in every recursion prompt

**Amphora** (vocabulary seated `20260703.203712`) names the vessel. The **manifest** is its table of contents — each cargo row names **what** (Tilak mark), **proof** (SHA3-256 hex), and **path** (name in the tree). The seed parses **one row**:

```text
<mark> <sha3-256-hex> <name>
```

Example (illustrative marks):

```text
research 6047187a66560393a010859c0efc98207607a5a90a99639b0d13090e5e385330 external-research/20260703-200712_mysticetis-blocks-and-the-object-path.md
silo 36c7d27794069bf1d74d6818f4a7acd65f3def0961cd00778d5587aca401ae5c active-designing/20260703-200712_what-needs-ordering.md
log b2fb8e1bbd3110fd7c7b18ec429e978a3ab13f6331017b1c5a857a4784f02e5d session-logs/20260703-200712_pottery-and-suite-reissue.md
```

The parser **refuses damage whole** — wrong field count, uppercase hex, missing name — matching the Amphora wreck rule at the line grain.

### What the seed does (mechanically)

[`crossing_manifest_seed.rye`](../work-in-progress/20260703-202312_crossing_manifest_seed.rye) exports one function's behavior via tests today:

- **`parse_manifest_line`** — split exactly three space-separated fields
- **Bounds** — mark ≤ 32 chars, name ≤ 200 chars, digest exactly 64 lowercase hex chars (SHA3-256 working tier per resins law)
- **Invariants** — TAME style: assert pre/post conditions; no silent partial parse
- **Tests** — well-formed line parses; damaged lines return `InvalidFormat`

It does **not** yet: read a file, hash bytes, verify digest against disk, build a zip, or sign. Ring one is **parse + verify**; emit comes later.

### Purpose in the larger crossing

```text
Shoulder line (human prose)
Parent mark (git tip)
Manifest (many lines of mark + digest + name)  ← this seed
Seal (Amber, ring two horizon)
Stamp (Kumara, horizon)
Vessel bytes (zip or amphora file)
```

Dogfood path: the loop's own nightly packet lists its cargo in manifest form so prompts stop repeating zip discipline — the vessel carries the law.
